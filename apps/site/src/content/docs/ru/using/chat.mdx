Чат в комнате — боковая панель для текста, реакций, опросов и изображений без выхода из звонка.

## Открытие и отправка

1. Откройте панель **chat** из контролов встречи.
2. Отправьте текстовое сообщение — другие участники его получат.
3. Открытие или закрытие панели **не должно** убивать tracks камеры или микрофона.

Группировка сообщений и метки времени должны выглядеть корректно даже при burst-отправке.

## Emoji

- **Реакция** на сообщение — обе стороны обновляют счётчики реакций.
- **Emoji picker** в композере вставляет emoji в черновик.

## Опросы

1. Создайте **poll** из чата.
2. Все видят bubble опроса.
3. Участники **голосуют**; результаты обновляются (модалка результатов, если UI её даёт).

## Изображения и файлы

- Прикрепите изображение или файл в пределах лимитов сервера.
- Получатели могут открыть **lightbox** для изображений.
- Большие payload идут chunked-путём, чтобы контент не отбрасывался молча.

Операторы настраивают квоты загрузки и storage под `chat` в [Конфигурации](/ru/docs/getting-started/configuration) (`maxUploadBytesPerUser`, disk/S3 upload settings).

## Retention (client-side)

LiveKit не хранит data-channel чат на сервере как отдельный chat-продукт. Bedrud применяет **client-side** подсказки retention:

| Настройка | По умолчанию (типично) | Смысл |
|-----------|------------------------|-------|
| `chat.maxMessageCount` | 10000 | Потолок сообщений в памяти / session storage |
| `chat.messageTTLHours` | 2160 (90 дней) | Обрезка по возрасту |

Env overrides: `CHAT_MAX_MESSAGE_COUNT`, `CHAT_MESSAGE_TTL_HOURS`.

После **перезагрузки mid-room** история следует этой product-политике (restore сессии или чистый старт) без краша.
