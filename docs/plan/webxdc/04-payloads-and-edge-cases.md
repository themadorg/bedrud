| Topic | Purpose | Reliability | Body |
|-------|---------|-------------|------|
| `webxdc` | Status updates (`sendUpdate`) | reliable | JSON envelope + serial metadata |

## Official app-facing limits

| Limit | Default | Host must |
|-------|---------|-----------|
| `webxdc.sendUpdateInterval` | 10000 ms | Expose; delay/coalesce faster `sendUpdate` |
| `webxdc.sendUpdateMaxSize` | **128000** bytes | Expose; reject larger **serialized** `update` object |
| Realtime message size | 128000 bytes | Reject larger `Uint8Array` |

Additional host protections (not in app API but required for safety):

| Limit | Suggested |
|-------|-----------|
| Status log depth | last 500 updates or 2 MiB per appId |
| Snapshot size | ≤ 512 KiB over DC (or use server log) |
| Concurrent open apps | 1–3 per client |
| `info` / `summary` / `document` display | truncate ~50 / ~20 / ~20 chars |

## Status update object (app → host)

From `sendUpdate`:

```ts
type WebxdcSendUpdate = {
  payload: string | number | boolean | unknown[] | Record<string, unknown> | null
  info?: string
  href?: string // relative only, e.g. "index.html#about"
  document?: string
  summary?: string
  notify?: Record<string, string> // selfAddr → text; "*" catch-all
}
// descr argument deprecated → ""
```

Rules:

- `payload` MUST NOT be `undefined`.
- Non-JSON-serializable values skipped (no raw `ArrayBuffer` / `Blob` in payload — apps use base64).
- `href` must be **relative** (reject `https:`).

## Received update object (host → app listener)

```ts
type WebxdcReceivedUpdate = {
  payload: unknown
  serial: number // > 0; gaps allowed
  max_serial: number
  info?: string
  href?: string
  document?: string
  summary?: string
  notify?: Record<string, string>
}
```

## LiveKit status envelope (bridge ↔ bridge)

```ts
type WebxdcStatusWire = {
  v: 1
  kind: "status"
  appId: string
  /** Transport hint only — receivers prefer LiveKit participant.identity for abuse logs */
  senderHint?: string
  serial: number
  ts: number
  update: WebxdcSendUpdate
}
```

```ts
type WebxdcControlWire = {
  v: 1
  kind: "control"
  appId: string
  action: "close" | "requestSnapshot" | "snapshot"
  /** snapshot: compact list of status updates or opaque blob */
  snapshot?: WebxdcStatusWire[]
}
```

### Validation

| Check | Action |
|-------|--------|
| Unknown `v` / kind | Drop |
| Missing `appId` | Drop |
| Serialized size > max | Drop |
| `href` absolute URL | Drop or strip |
| `control: close` | Apply only if sender is mod/owner (RBAC) |
| Nested snapshot bombs | Reject |

### Identity

- Do **not** trust app payload for “who is admin.”
- `selfAddr` is **per-app opaque**; generated by host, not equal to email.
- For meeting chrome “who sent,” use LiveKit display name + trusted identity.

## Realtime wire

Prefer minimal header + raw bytes to stay under 128000:

```ts
// Example: first N bytes JSON header length-prefixed, or separate topic filter by appId only
type WebxdcRtWire = {
  v: 1
  appId: string
  // data follows as binary in same packet or second chunk
}
```

No serial, no snapshot, no persistence.

## postMessage (iframe ↔ parent)

```ts
// iframe → parent
type ToHost =
  | { channel: "bedrud-webxdc"; type: "sendUpdate"; appId: string; update: WebxdcSendUpdate }
  | { channel: "bedrud-webxdc"; type: "rtSend"; appId: string; data: ArrayBuffer | number[] }
  | { channel: "bedrud-webxdc"; type: "rtJoin" | "rtLeave"; appId: string }
  | { channel: "bedrud-webxdc"; type: "ready"; appId: string }

// parent → iframe
type ToApp =
  | { channel: "bedrud-webxdc"; type: "statusUpdate"; update: WebxdcReceivedUpdate }
  | { channel: "bedrud-webxdc"; type: "rtData"; data: ArrayBuffer }
  | { channel: "bedrud-webxdc"; type: "init"; selfAddr: string; selfName: string; sendUpdateInterval: number; sendUpdateMaxSize: number }
```

Rules: exact `event.source`, exact `event.origin`, never `targetOrigin: "*"`, bind `appId` to iframe instance.

## Edge cases

### Instance model

- One iframe per `appId`; same `.xdc` file opened twice → two instances.
- Storage isolated per instance (messenger MUST).

### Late joiner (status only)

1. Request snapshot / load server log.
2. Replay via `setUpdateListener` from `serial=0` or stored cursor.
3. Realtime channel: empty until peers send again.

### Delivery (faq/storage)

- Assume loss and reorder; apps should use CRDT/full-state patterns.
- Host still does best-effort reliable DC + optional server log.

### Moderator close

- RBAC on receiver; tear down iframe; `rtLeave`; retain or wipe status log per product choice.

### `info` + `href`

- Show info in activity feed (plain text).
- Click → ensure app started → navigate webview to `href` relative to app root.

### External links

- If app uses `<a href="https://…">`, intercept: confirm full URL + privacy warning (spec 1.3). Default deny without confirm.

### `notify`

- Only notify local user when `notify[selfAddr]` or `notify["*"]` present.
- Never trust notify to execute HTML.

### Permissions (security-leaning defaults)

| Action | Who |
|--------|-----|
| Upload `.xdc` | Owner / mod |
| Start / open instance | Room members / guests per policy |
| sendUpdate / realtime | Anyone with app open |
| Force close | Owner / mod |
| Delete package | Owner / mod / admin |
